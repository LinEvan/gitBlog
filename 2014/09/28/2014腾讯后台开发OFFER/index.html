2014腾讯后台开发OFFER

By LinEvan

9月 28 2014 更新日期:12月 17 2014

文章目录

19号腾讯HR面试结束后，我一直等到25号上午才接到HR电话，虽然等待的过程有点煎熬，但我还是如愿拿到了腾讯WXG后台的OFFER。至于百度的OFFER，因为我实在不想去北京也只能拒了。
简单谈谈腾讯的招聘吧。4月份腾讯招实习生时，我投了后台开发，但由于准备不充分直接在笔试就跪了。经过几个月的准备，基础牢固很多，但对于腾讯的笔试我还是没什么自信。腾讯考察的范围太广了，数据库、编译原理、设计模式、计算机基础、网络编程、操作系统基本什么都考，虽然考得不深，但有些考得也挺偏的，这么短的时间也没法准备。而且12号阿里面试，13号腾讯笔试，我既要准备面试又要准备笔试，相当的吃力。由于阿里第一面就跪了，所以后来就可以好好准备腾讯招聘了。
今年腾讯的笔试题很简单，25道选择题，还有3道大题，考察的大都是C++和网络编程，原题如下2014腾讯研发笔试题-南京站。刚好C++和网络编程我比较擅长，所以很轻松就过了笔试。 16号进行一面。面试时间40分钟左右。刚进去他让我自我介绍，然后问我有没有拿到其他公司的OFFER，我说了拿了百度研发的OFFER。顿时感觉面试官精神一阵，然后他问我说：“那你应该更愿意去百度吧？”我笑笑说，我家是广东的，想回家。然后主要问了以下几个问题：
（1）比较擅长的是什么？我回答了C++和数据结构，很奇怪面试官也没具体问C++和数据的结构的知识。
（2）多线程与多进程的区别？哪个更好用，哪个更稳定？
（3）能不能设计一个框架来判定和避免僵尸进程。
（4）进程间通信的方式有哪些？然后问笔试题中进程间通信用哪个方式比较好？如果是在一个Linux服务器而不是分布式服务器上用哪些方式
（5）有没有开源项目经验，是否看过开源代码？
多线程和多进程了解得比较多，项目也都用过，所以问题不大。僵尸进程了解太浅，所以回答得不是很好，开源项目也怎么接触过。总体表现一般般，但由于我有丰富的项目经历，又有百度OFFER撑着，所以面试现场官直接说我可以过一面了。 18号进行二面。面试时间也是40分钟左右。我一进去本来想递简历给他，他笑笑说我这边有你的简历。我才知道腾讯是按照你填简历时报的Group来给你安排面试官，所以面试官都是定的，而且通知面试的时间跟你正式面试的时间差不多，有时还会提前面试，单从这一点就秒阿里几条街。阿里面试你不等个一两个小时他是不会让你面的，而且面试官都是随意安排的，有时C++面试官来面Java，Java面试官来面C++。好了，回归主题：
（1）首先让自我介绍。中间打断我两次，问了Dian团队，还有华科的为什么要来东大，囧，早知道会问这问题了
（2）问static关键字（讲了作用于函数、变量，以及类的变量和成员函数的情况），面试官还问使用static有什么限制，刚开始没想出来，经提示才知道是可重入函数下static变量的互斥问题）
（3）问了TCP/IP四次握手，还有状态迁移图
（4）实现多线程时用了什么多线程库？有哪些多线程库
（5）IO实现的几种方式，说说特点（select、poll、epoll），还提到了线程池概念（惊群效应）
（6）实现一个算法：n个数组成圆圈，从第k个数开始数，到第m个数出列，然后从下一个开始再到第m个数出列，循环到所有数输出为止。面试官直接说用数组实现，很简单，几分钟就写完了。
二面很简单，基本都答上来，面试官很一直夸我基础扎实。二面面完我知道自己肯定能过。
19号下午进行HR面。面试官是个女HR，长得还挺漂亮的，名字就不说了，所以我的面试积极性一下子提升了不少。HR面基本什么都面，本科生活、研究生生活、家庭情况、感情经历（HR比较八卦，连为什么分手都问）、优缺点等等，聊得也比较开心，最终面了一个多小时，估计是当天面HR面最久的。面试完我感觉问题不大了，回来微信状态一查也改为“已完成所有面试”，然后开始等OFFER了。 总体来说，腾讯的笔试很基础，但考得比较广。技术面试主要也是考察语言基础、网络编程、操作系统、算法等等，对于项目考察得不多。HR面的话各个公司应该都差不多。比较想吐槽就是腾讯发OFFER很不干脆。阿里是当天面完，第二天就通知全部通过的人签两方；腾讯由于部门繁多，手续也很复杂，所以一直不给个信，微信状态也不靠谱，这个等待的过程很让人抓狂，希望腾讯能提前定好招人计划，不要搞什么备胎策略，发不发OFFER直接给个痛快话。
腾讯定了之后，我的求职也基本结束了，接下来好好练下lol，进了鹅厂带妹子撸。
